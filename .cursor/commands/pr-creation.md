# pr-creation

## 目的
完了した作業内容をPull Requestとしてまとめる際に、Plan/Taskとの紐付けやテスト結果を整理し、PR本文テンプレートを生成する。

## 引数
`/pr-creation <docs/tasks/.../task.md または 変更概要MD>`

## 実行フロー
1. 引数で指定されたTask/概要ファイルを読み込み、関連するPlan、完了ステップ、検証結果を抽出する。
2. 現在の作業ブランチ名とターゲットブランチ（通常`main`）を確認し、未コミット変更や未Pushがないかチェックを促す。
3. PR本文のドラフトを生成する。推奨テンプレ:
   ```
   ## Summary
   - この変更の目的/背景（Plan参照）
   - 主な実装内容

   ## Testing
   - [ ] `make test-backend`
   - [ ] `npm run test`

   ## Docs
   - Source Plan: ...
   - Task: ...
   ```
   必要に応じてチェックボックスを✔で埋め、不要な項目は削除する。
4. 追加でレビュー観点やリスク、フォローアップIssueを記載する欄を用意し、ユーザーに追記を促す。
5. 生成した本文とともに、`git push origin <branch>`→GitHubでPR作成の手順を案内する。
6. 要求があれば`gh pr create`コマンド例を示す。
7. すべて日本語で出力し、最後に次のアクション（例: 「レビュアーをアサインしてください」）を提示する。

## 注意事項
- Taskが未完了またはVerification未実施の場合はPR作成を保留し、必要な作業を列挙する。
- 変更範囲が大きい場合はハイライトをまとめ、スクリーンショットやログ添付が必要か確認する。***
