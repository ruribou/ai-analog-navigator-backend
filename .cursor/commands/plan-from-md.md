# plan-from-md

## 目的
`docs/plans/`にある計画Markdownを読み込み、対応する実装方針ドキュメントを`docs/tasks/`に生成する手順をエージェントへ指示します。

## 引数
`/plan-from-md <docs/plans/.../plan.md>`

## 実行フロー
1. 指定されたパスが`docs/plans/`配下の既存Markdownか検証する。存在しない場合はユーザーにパス修正を依頼する。
2. 計画Markdownを要約し、ゴール・非ゴール・成功基準・未決事項を洗い出す。
3. `docs/tasks/`に配置する新しい実装方針ドキュメント案を作成する。推奨レイアウト:
   ```
   # Task: <計画タイトル>
   - Source plan: <plan path>
   - Status: Draft

   ## Summary
   ...

   ## Proposed steps
   1. ...

   ## Verification
   - ...
   ```
4. 新しいファイル名は日付+テーマ（例: `docs/tasks/2025-11-foo.md`）で提案し、ユーザーに保存可否を確認する。
5. 承認後、ファイル作成または編集方法を提示し、必要であれば自動編集（Agentモード）を行う。
6. 最後に生成した実装方針の要約と次のアクション（例: Accept待ち）を報告する。

## 注意事項
- 計画ドキュメントに定義された判断を尊重し、欠落情報はユーザーへ質問する。
- 既存の`docs/tasks/`ファイルを上書きする場合は必ず確認を取る。
- すべて日本語で回答する。
